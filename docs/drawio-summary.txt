# target-architecture-gap.drawio 摘要

更新时间：2026-06-30

本 drawio 文件用于说明 FAMS 当前阶段的用户体验优化目标、当前架构与目标架构差异、体验解释层技术路径、红利低波用户路径、组合回测用户路径、ChatBox/PI AgentCore 业务入口、开发及验收计划、项目里程碑、验收门槛和出门条件。页数固定为 8 页，不超过 8 页。图内不使用无法落地的泛化模块，每个关键节点都绑定到真实页面、API、服务、数据/缓存、体验组件、AgentCore 适配器或审计 artifact。

当前阶段仍是正式交易 release 前置评审，不是正式交易 release。体验优化只能降低理解成本，不能隐藏数据缺口、模型不足和交易锁定。`dataTrustGrade=INSUFFICIENT` 时只能研究观察；`calculationAuditStatus=deterministic_replay_only` 只证明同一输入下公式复算一致，不证明模型有效或策略可交易。

## 颜色规则

- 灰色：已经实现或本阶段已有基础能力。
- 黄色：需要修改、补齐或状态不达标的能力。
- 橘黄：需要新增的体验解释、正式评审或 release 前置能力。
- 红色：交易硬边界或禁止事项。
- 蓝色：用户路径。
- 绿色：阶段输出、验收结果或允许出门状态。
- 紫色/蓝紫色：体验解释层，包括普通模式、专业模式、结论卡、术语解释和下一步操作。

## 页面结构

1. **目标体验与用户路径**：普通用户默认看状态摘要、普通话结论、下一步和禁止动作；专业用户可进入完整指标、validation 和审计 artifact。
2. **当前架构与目标架构差异**：按前端、API、服务、数据、审计/Gate 五层展示“当前基础 -> 需修改 -> 需新增 -> 硬边界 -> 用户体验结果”的强关联路径。
3. **体验解释层架构**：展示 `ExperienceModeToggle.tsx`、`PlainLanguageHelp.tsx`、`DividendLowVolDecisionCard.tsx`、`PortfolioBacktestSummaryCard.tsx`、`StrategyComparisonExplainer.tsx` 如何接入现有页面、API 和审计。
4. **红利低波用户路径**：展示状态摘要、Top 候选卡、买卖观察区间、数据可信、风险提示、人工计划草案和专业证据展开路径。
5. **组合回测用户路径**：展示策略选择、多时间段收益曲线、收益来源解释、数据等级、模型有效性、交易阻断和人工计划草案。
6. **开发及验收计划**：把 UX-1 到 UX-6 映射到实现实体、用户效果、审计产物和验收命令。
7. **里程碑与出门条件**：列出 M1 到 M4、当前可声明状态、当前不能声明状态、审计文档路径、截图验收和外部审计重点。
8. **ChatBox 与 AgentCore 业务入口**：说明 ChatBox 如何通过 `FamsChatBox.tsx / chat.ts / famsChatService / piAgentCoreAdapter` 串起业务查询、二次确认、Operation、页面跳转和交易阻断。

## 每页验收重点

- 第 1 页：普通用户能看懂系统当前能做什么、不能做什么、下一步做什么；专业用户能进入证据详情。
- 第 2 页：审核者能按前端、API、服务、数据、审计/Gate 五层看清当前架构如何演进到目标架构。
- 第 3 页：开发者能明确体验解释层只重组信息层级，不改变策略计算、数据可信判断和交易 gate。
- 第 4 页：用户能通过红利低波页面理解 Top 候选、买卖观察区间、风险提示、数据可信和人工计划草案边界。
- 第 5 页：用户能通过组合回测页面理解不同起止时间段收益曲线、收益来源、benchmark 资格、模型有效性和交易阻断。
- 第 6 页：开发者能按 UX-1 到 UX-6 执行体验降复杂度开发，并用桌面、平板、移动端截图和审计 JSON 证明普通用户可理解。
- 第 7 页：审核者能看懂里程碑、出门条件、不能声明的状态、待审计文档路径和高风险阻断。
- 第 8 页：审核者能看懂 ChatBox 当前 v1、PI AgentCore 后续目标、工具权限分层、二次确认、Operation 联动、交易阻断和审计产物。

## 风险与备选路线

当前文档允许后续自动化开发继续推进 UX 优化，但不隐藏以下真实风险：

- 体验简化可能误导用户忽略数据缺口、模型不足或交易锁定。
- 普通模式如果只展示结论而不展示可信度，会形成虚假确定性。
- 专业模式如果被移除，审计和复核能力会倒退。
- 免费数据源、proxy benchmark 和 deterministic replay 仍不能作为正式交易 release 证据。

推荐路线：

```text
普通模式默认展示结论和下一步
+ 专业模式保留完整证据
+ 所有 blocker 永久可见
+ 截图和审计 JSON 双验收
```

该路线可以支撑 ordinary-user-ready 和 formal-review-ready；不能直接声明正式交易可执行。
