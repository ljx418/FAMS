## 05 模型验证与 Release Gate
Nodes: 9; Edges: 0

Nodes
- 模型验证与 Release Gate：长周期回测必须经过 OOS、walk-forward、参数和分组稳定性
- 已具备评审材料 / portfolioBacktestFormalReviewReady=true / manualDraftReady=true / paperSandboxReviewReady=true / interactiveStrategyBacktestReady=true / longHorizonRealDataBacktestReady=true
- OOS 样本外验证 / 训练期与样本外收益、胜率、尾部亏损必须对比。 / 样本外收益或超额收益为负时，不得 formal passed。
- Walk-forward 验证 / 至少 6 个窗口。 / 通过窗口比例不足或单窗口通过时，保持 warning/insufficient。
- 参数敏感性 / 月度/季度再平衡、费用、滑点、分红现金/再投资等邻域重放。 / 不得用单点参数通过替代稳定性。
- 分组稳定性 / 行业、市场状态、流动性、市值分组。 / 分组样本不足时保持 insufficient，不升级 passed。
- 人工签核 / manualSignoffAudit.status=missing / 数据、模型、风控、合规、最终 release 缺一不可。
- Release Gate / releaseGateAudit.status=blocked / 任一 gate 未通过时：formalTradingUnlocked=false / canCreateOrder=false / orderCreateAllowed=false。
