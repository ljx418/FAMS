项目背景：
FAMS 是个人/家庭投资资产管理与研究决策系统。系统目标不是让 AI 直接买卖，而是：
1. 构造可靠事实集；
2. 由确定性规则、行情数据、回测证据、持仓约束生成可审计结论；
3. AI 只负责解释、归纳、生成研究报告或交易计划草案；
4. 最终买卖必须由用户人工确认。

当前系统已具备：
- 资产与持仓账本；
- 股票、ETF、基金、债基、黄金、现金资产类型；
- market_bar_raw；
- market_bar_canonical；
- market_data_coverage；
- market_feature_daily；
- provider health；
- Operation 状态机；
- AI 选股；
- strategy tournament；
- validation evidence artifacts；
- PositionAdviceFactSet；
- PositionAdviceEngine；
- ADD / REDUCE 当前仍被 validation_evidence gate 阻断。

本次目标：
实现 FAMS Analysis Advice Core，即：
Investment Valuation + Expected Return Distribution + Trading Discipline + PositionAdvice Adapter + Advice Replay Validation + Intervention + Model Tuning。

核心边界：
1. LLM 不直接决定 BUY / SELL / ADD / REDUCE。
2. 没有 evidenceRefs 的建议不能作为交易建议。
3. 数据不足、provider 冲突、回测验证不足时，只能输出 OBSERVE / HOLD / NO_ACTION / 风险提示 / 研究结论。
4. Validation Evidence 未通过时，不允许 formal ADD / REDUCE。
5. 所有模型输出必须包含 modelVersion、dataVersion、runId、evidenceRefs、blockedReasons。
6. 所有 Replay 必须使用 reviewDate 当时可见数据，禁止未来函数。
7. 所有人工干预必须 append-only，不允许覆盖历史结果。

====================
Phase 1：价值评估模型 MVP
====================

实现 services：

1. EvidenceGateService
职责：
- 检查 asset identity；
- 检查 market_data_coverage；
- 检查 provider confidence；
- 检查 market feature freshness；
- 检查 validation evidence；
- 输出 EvidenceGateResult。

硬规则：
- asset type unknown => blocked；
- latest price stale or failed => blocked 或 partial；
- provider conflict severe => blocked；
- validation evidence failed => formal ADD blocked；
- cash => no valuation needed。

2. InvestmentValuationService
根据 assetType 分发：
- StockValuationModel
- EtfValuationModel
- FundValuationModel stub
- BondFundValuationModel stub
- GoldValuationModel
- CashNoActionModel

3. StockValuationModel
MVP 分数：
- valuationScore：PE / PB / 历史分位 / 行业分位，缺失时 partial；
- qualityScore：ROE / 毛利率 / 经营现金流 / 资产负债率；
- growthScore：营收增速 / 净利润增速 / 扣非增速；
- financialRiskScore：高负债、现金流恶化、业绩恶化、事件风险；
- timingScore：trendScore / momentumScore / relativeStrength / volatility；
- portfolioFitScore：当前仓位、核心/卫星、集中度、现金比例；
- evidenceQualityScore：来自 EvidenceGate。

4. EtfValuationModel
MVP 分数：
- underlying index quality；
- index valuation percentile；
- liquidity；
- fee if available；
- portfolio fit；
- data missing => partial。

5. GoldValuationModel
不要套股票估值。
输出：
- macroSupportScore；
- hedgeValueScore；
- portfolioDiversificationScore；
- timingScore；
- portfolioFitScore；
若宏观数据缺失，只能输出观察，不输出确定性结论。

6. CashNoActionModel
现金不生成投资建议，只输出 cash_no_action / no_action。

7. PositionAdvice Adapter
把 InvestmentValuationOutput 映射为：
- valuationMultiplier
- qualityGrowthMultiplier
- riskPenaltyMultiplier
- portfolioFitMultiplier
- evidenceConfidenceMultiplier
- validationGateMultiplier
- targetWeightMultiplier
- allowedActionHints
- formalTradeActionAllowed

规则：
- evidence insufficient => allowedActionHints 不包含 ADD；
- validation evidence failed => formalTradeActionAllowed = false；
- cash => only NO_ACTION。

数据库：
新增或迁移：
- valuation_runs
- valuation_factsets
- valuation_scores

如果项目当前使用 Prisma，请新增 schema 和 migration。
如果当前仍兼容 SQLite，注意避免使用 SQLite 不支持的特性。

验收：
1. 股票、ETF、黄金、现金分别走不同模型。
2. 现金输出 cash_no_action，不输出交易建议。
3. 基本面缺失时 status = partial 或 insufficient。
4. 所有分数有 reasons / warnings / evidenceRefs。
5. validation evidence failed 时 formalTradeActionAllowed = false。
6. 不允许 LLM 直接决定 action。
7. 新增单元测试：
   - stock with sufficient data
   - stock with missing fundamentals
   - etf with missing index valuation
   - gold with missing macro facts
   - cash no action
   - validation failed blocks formal ADD

====================
Phase 4：Advice Replay Validation
====================

实现 AdviceReplayEngine。

新增表：
- advice_replay_runs
- advice_replay_slices
- advice_decision_logs
- simulated_trades
- replay_portfolio_snapshots
- probability_calibration_results

支持：
1. Fixed Start Replay：
   - 3 个月前；
   - 2 个月前；
   - 1 个月前；
   - 每周复盘到今天。

2. Rolling Start Replay：
   - 最近半年每周作为起点。

Replay modes：
- gate_strict
- research_only
- buy_hold
- benchmark

每周流程：
1. 使用 reviewDate 当时可见数据；
2. 运行 InvestmentValuationService；
3. 运行 PositionAdviceEngine；
4. 运行 TradingDisciplineEngine；
5. 运行 validation gate；
6. 生成 simulated trade；
7. T+1 成交；
8. 扣手续费、印花税、滑点；
9. 更新 replay portfolio snapshot；
10. 记录 decision log。

禁止：
- 使用未来 K 线；
- 使用未来财报；
- 使用未来新闻；
- 使用今天才知道的数据修正历史判断。

验收：
- 可从 3m/2m/1m 起点回放到今天；
- 输出最终市值、累计收益、最大回撤；
- 输出 FAMS vs BuyHold vs Benchmark；
- 每笔 simulated trade 可追溯 decision_id；
- Gate-Strict 下 validation failed 不执行 formal ADD / REDUCE；
- Research-Only 可以记录理论动作，但必须明确标记不可作为正式交易建议。
